# Checkpoint Guidelines

To ensure the "Step-by-Step" (步步为营) principle, use these checkpoints for user alignment:

## 1. Design Checkpoint
Trigger this **before** starting implementation when:
- Defining new API endpoints or internal interfaces.
- Designing database schemas or key data structures.
- Proposing a significant architectural change.

## 2. Implementation Checkpoint
Trigger this **during** implementation when:
- A core module or complex logic block is completed.
- You encounter an unforeseen technical hurdle that requires a pivot.
- You've completed about 50% of the planned task list.

## 3. Release Checkpoint
Trigger this **after** implementation but **before** merging:
- When the Pull Request is ready for final review.
- After all tests have passed.
- When you are ready to "deploy" or "activate" the changes.
